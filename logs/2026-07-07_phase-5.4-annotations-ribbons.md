# Phase 5.4 — Loop ribbons, publication annotations, region camera focus, intro, dual-register content

**Date:** 2026-07-07

The final Phase-5 stage: the content layer over the Approach-B unwind engine.
Phase 5 is complete. Verified in both isolated harnesses AND on the live
site (user-cleared for this stage's final pass).

## What was done

- **`coil-anchors.ts` (+4 tests):** per-region compact anchor (the region's
  CENTER BEAD on the rim — a ~0.86-turn region's centroid collapses toward
  the cluster axis, useless as a marker) and open anchor (the unwound arc's
  centroid), lazy-memoized like arbor-anchors.
- **Growth default `regionGap 0.33 → 0.375`:** at 6 turns, 0.33 was exactly
  2.0 revolutions — both loci at the SAME azimuth ~1 unit apart (overlapping
  labels, twin unwind bearings). 0.375 = 2.25 revolutions → ~82° apart on
  the rim. Region marking never moves compact placement, so the blessed
  silhouette is bitwise untouched; only the locus glows moved.
- **Loop ribbons:** `buildRibbonGeometry`/`writeRibbonGeometry` (both
  regions' `loopArcPairs` arcs in ONE mesh, static topology, rewrite-in-place
  per unwind tick from the same node state as the beads/threads; arcs bow
  outward off the cluster axis + lift). `coil-ribbon` shaders: linker-style
  profile/fog + strand-style traveling packet over `aArcT` (per-arc phase),
  whiten at the front; visibility = region match × `smoothstep(0.35, 0.9,
  uUnwind)` — the streams bloom in the LATER half of the opening. JS hides
  the mesh entirely while compact. Ribbon look params + leva sliders.
  +1 draw call (coil scene = 3).
- **Region camera focus:** `regionFocusPoseFor` in camera-focus.ts (+2
  tests) — bearing from the compact anchor's rim direction (the open
  centroid sits near the cluster axis; a radial through it is
  ill-conditioned), target = open centroid, stand-off 17 / lift 2 / fov 45.
  Controller: a parallel held-pose block chained after the branch block that
  **rides `unwindBlend`** — geometry unwind and camera pivot are one
  gesture, no second tween; region switches pass through blend 0 exactly
  when the held pose swaps (pop-free by construction).
- **`CoilAnnotations.tsx`:** ArborAnnotations pattern (gsap-ticker
  projection, envelope 0.465→0.50 / 0.545→0.565, Esc, first-close hint,
  clamping, side logic with `hoveredRegion` freeze — new store field,
  +1 test). Coil deltas: the anchor RIDES `lerp(compact, open, blend)` (a
  shown-region tracker mirrors CoilMesh's release-then-focus sequencing;
  blend joins the skip-work guard), and **screen-space label de-collision**
  (min 64 px separation, symmetric push) — two loci on one compact cluster
  can project coincident at view angles aligned with their chord, found
  garbled on the live site and verified overlap-area 0 across the window.
- **`CoilIntro.tsx`:** ArborIntro pattern, window 0.435→0.46 / 0.475→0.50
  (clears before annotations), painted lens, blue-register title (sans).
- **Dual-register `ChromatinContent`:** runways (140vh arrival / 130vh
  index) + intro + annotations + `.chromatin-doc` (hidden under WebGL, badge
  re-homed inside). CSS: chromatin added to the WebGL-reveal selector lists
  (section transparent, atmospherics off), runway/doc/intro/annotation
  blocks mirroring the arbor's, `.coil-entry__*` card classes. Publication ↔
  region mapping is array order (2 entries, 2 loci; commented).
- **Descent harness:** `data-webgl='active'` + `data-motion` reflected at
  bootstrap (the dot-pulse CSS keys on the latter — found via a
  pixel-stability failure that was a harness gap, not an app bug); mounts
  `CoilIntro` + `CoilAnnotations`, so the full interaction verifies in
  isolation. Placeholder lorem body added to the band's section prose.
