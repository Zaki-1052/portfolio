# 2026-07-05 — Phase 4.1 + 4.2 harness: Arbor generator, tree mesh, tuning panel

Phase 4 plan approved and saved as `docs/p4-plan-arbor-scene.md`. Stage 4.1 built and
gated; the 4.2 tuning harness (live-params channel + leva panel) built in the same
session so the user's 4.1 eye-pass doubles as a slider session. Stills captured.
Language convention holds: neutral geometric/botanical vocabulary only
(arbor / trunk / limb / strand / tip).

## What was done

- **`src/utils/arbor-generator.ts` (+11 tests, TDD)** — pure seeded branching generator
  (mulberry32): trunk → 3 limb spines at unequal azimuths with per-limb spread/reach
  jitter → fine strand recursion from spine tips AND probabilistic side sprouts along
  the spines (`sideSproutRate` — pure tip-recursion read as tufts on sticks). Nodes
  carry `{position, radius, parent, limb, level, region, t}`; `t` = normalized
  cumulative path length (drives the solid→luminous gradient).
- **`src/scales/cellular/arbor-params.ts`** — growth+look param set, presets
  (`sparse`/`dense`/`gnarled`), `ARBOR_ORIGIN [2,-14,-28]`, `applyArborTrunkLook`/
  `applyArborGlowLook` mappers (shell-params pattern).
- **`src/scales/cellular/arbor-anchors.ts` (+3 tests, TDD)** — `BRANCH_ANCHORS` from
  per-limb deepest tips + origin; single source of truth for 4.4 cards/focus poses.
- **`src/scales/cellular/arbor-geometry.ts`** — three builders: merged tapered-tube
  sweep with parallel-transport frames + Catmull-Rom spine subdivision (SUBDIV 3 —
  joints don't facet); camera-facing ribbon quads for strands (billboard axis in the
  vertex stage; GL lines can't taper); tip Points. Per-vertex seeds hash from NODE
  indices so adjacent ribbon segments + tip sprites sway as one connected piece.
- **Shaders** (6 files under `src/scales/cellular/shaders/`) — trunk vert/frag in the
  shell's craft register cooled to rose (hemisphere + wrapped key + tight fresnel rim,
  fbm bark relief/mottle, manual exp2 fog, uOpacity dim, focus/hover uniforms ready
  for 4.4); strand/tip vert/frag additive premultiplied glow, white-shifted toward
  tips so the periphery crosses the band's bloom threshold, distance-extinction fog.
- **Materials** — `arbor-trunk-material.ts`, `arbor-glow-material.ts` (drei
  shaderMaterial, highp on trunk).
- **`ArborMesh.tsx` / `CellularScene.tsx`** — assembly + per-frame uniform writes
  (fog mirror, uTime, uPixelScale); registry swap in `scene-manager.tsx`
  (`cellular: CellularScene`, key `'arbor'`); `CellularVoidStub.tsx` deleted.
- **`cellular-preview.html` + `src/dev/cellular-preview.tsx`** — isolated dev preview
  (params: seed/preset/rx/ry/spin/z/fov/dpr(2)/sway), real PostFX parked at depth
  0.36 (band register), `window.__preview` handle for scripted GL verification.
- **`render-loop.ts`: `setAmbientRendering` → refcounted `acquireAmbientRendering()`**
  — the old boolean let whichever overlapping scene unmounted first freeze the
  survivor's idle drift (latent bug exposed by having a second scene). TissueScene
  migrated.
- **4.2 harness (same session)** — `arbor-live-params.ts` (subscribable override
  channel: growth edits need a geometry-rebuild push, not a uniform poll);
  `src/dev/arbor-dev-tools.tsx` (`useArborControls` + `<ArborDevTools/>`, 'arbor
  growth' + 'arbor look' folders + preset buttons, shell-dev-tools pattern);
  `ArborMesh` subscribes and rebuilds (~1ms) / re-applies uniforms on change and

  mounts `<ArborDevTools/>` lazily beside the other three panels.
- **Site smoke test** — overture plays, `#cellular` hash jump lands in-band, both
  scenes coexist over the handoff, zero console errors, 59 FPS (Playwright).
  The shell's gold interior currently dominates the band's opening — that's 4.3's
  rose fog handoff / reveal work, noted, not a 4.1 defect.

## Decisions made

- Fresnel on thin cylinders needs a much tighter power than the shell (5.5 vs 3.5) +
  low base weight — most of a tube's surface is near-grazing, so a broad rim tints
  all the bark pink (first render's failure mode).
- Two small glow frags instead of the planned shared one — a shared frag branching
  between `gl_PointCoord` and a ribbon varying risks the same UB family as the
  brown-ring bug. ~10 duplicated lines, zero risk.
- Strand/tip sway seeds are per-NODE (not per-edge) so chains stay watertight and
  sprites track their strand ends exactly.
- Review stills saved at repo root (`arbor-0*.png`) — disposable after the eye-pass.
