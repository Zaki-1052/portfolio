# Phase 3.5 — Overture, Spiral Descent, and Shell Resculpt

> Language convention (hard requirement): all code, comments, docs, and commit messages produced in this phase use only neutral geometric/design vocabulary — "the shell," "the form," "ridges," "cleft," "lobes." Same convention the existing shader source already follows.

## Context

Phase 3 shipped a technically solid first scene, but it reads as an *object on a page*, not the journey the docs describe — the user summarized it as "looks like a vase," and the audit agrees:

- **SPEC §7's overture** (typed intro → cinematic push-in → scroll unlock) was deferred to Phase 6.1 and doesn't exist at all. But its final frame IS the scroll journey's opening frame — deferring it means authoring the camera language twice.
- **SPEC §5's arrival phase** got collapsed: the visitor lands directly on hero text with the form as a static backdrop. The immersive beat was skipped.
- **The camera has never rotated.** All 5 keyframes have identity quaternions — a straight head-on z-dolly with zero parallax. SPEC §4 promises per-scale rotations/offsets.
- **The form itself** reads as a fluted vessel: rotationally symmetric vertical-teardrop silhouette + strictly meridian (pole-to-pole) ridge stripes + dead-center framing.

Phase 3.5 reworks all four together. User-locked decisions: **full overture now** (mono type, warm voice, placeholder copy); **spiral descent** arrival choreography with decaying approach speed; **pointer parallax**; **resculpt + immersive finale**. User's sculpt directives: oblong horizontal loaf (not vertical teardrop), wide flat base, central groove running front-to-back dividing two slightly-asymmetric halves, ridges that twist/loop/pack like tightly coiled strands instead of vertical stripes.

## Verified load-bearing facts

- The cleft is keyed purely on `|dir.x|` (`shell-shape.glsl:46,54`) — a great circle in the X=0 plane. Re-orienting the anisotropic scale (long axis → Z, flatten Y) makes the *same* formula read as front-to-back over the top. The resculpt is a rescale + notch/lobe removal, **not** a geometry rewrite.
- The meridian stripe axis is baked into `atan(dir.z, |dir.x|)` evaluated in **three places** that must stay consistent: `ridgeProfile` (`shell-shape.glsl:113`), `detailHeight` (`:136`), and the frag AA phase (`tissue-shell.frag.glsl:48`).
- The dissolve aperture is hard-coded to world +z on **two** axes: `front = smoothstep(-0.35, 0.7, normalize(vWorldPos).z)` AND `radial = length(vWorldPos.xy)` (`tissue-shell.frag.glsl:152-153`). Both must follow the authored approach axis.
- RD texture is sampled as a flat planar projection `vWorldPos.xy*0.04+0.5` (`:142`) — smears the moment the camera sees the flanks. Needs a dir-based equirect UV.
- The shell's self-fog uniforms (`uFogColor/uFogDensity`) are set once and never updated — invisible today (density constant), visibly wrong once fog density becomes depth-driven. Fix required, not optional.
- Per-section scroll runway is already freely variable: `rawProgressToDepth` remaps arbitrary DOM heights onto fixed canonical bands, with ResizeObserver + fonts.ready re-measure. Only `ScaleSection`'s `full` (pinned 100vh) needs changing.
- No intro infrastructure exists anywhere (no component, no store flag, no scroll lock). `DepthIndicator` has live keyboard/click scale-jump nav that must be guarded during the overture.
- `content/sections/brain.md` has an unused `subtitle` frontmatter field — the arrival block's one-liner, already authored.
- `frameloop="demand"`: any new animation (intro timeline, parallax spring, mote drift) must call `invalidate()` or ride the ambient ticker.

## Work packages (dependency order)

### WP0 — Camera keyframe schema + sampler rework
- `src/engine/camera-keyframes.ts`: replace `quaternion` with `target: Vec3` + `roll: number`; add `reducedAnchor?: boolean`. Generalize `tangent`/`hermiteAxis` to a field accessor (Hermite runs on position AND target). New pure `lookAtQuaternion(position, target, roll, worldUp)` (with parallel-up guard: if `|dot(fwd, up)| > 0.999` use secondary up) and `forwardFromQuaternion(q)`. Delete `slerp`. `sampleNearestKeyframe` derives quaternions the same way; `REDUCED_ANCHOR_KEYFRAMES = CAMERA_KEYFRAMES.filter(k => k.reducedAnchor)` becomes the reduced-motion track (~5 knots so snapping doesn't read as a 9-frame slideshow).
- Rewrite `CAMERA_KEYFRAMES` to ~9 knots (starter values, leva-tuned):

| depth | beat |
|---|---|
| 0.00 | establish — far + elevated, form ~15% frame height, heavy fog (anchor) |
| 0.03 | orbit begins — lateral sweep, descending |
| 0.06 | orbit + close — surface distances in geometric sequence (~0.65×/knot) so Hermite reads as decaying approach speed; knot placement, no new easing |

- `src/engine/camera-dev-tools.tsx`: per-knot leva fields become pos / target / roll(deg) / fov; bake prints the new schema.
- Update `camera-keyframes.test.ts`: schema migration, `lookAtQuaternion` identity + parallel-up guard (no NaN), anchor coverage spans [0,1].

### WP1 — Resculpt + ridge-field rework (shader-only; parallel with WP0; reviewed via tissue-preview at dpr 2)
**Base form** (`shell-shape.glsl`, `tissue-shell.vert.glsl`):
- `SHAPE`: elongate Z ~1.35× (long axis), flatten Y ~0.8×, X near baseline → horizontal loaf; the existing cleft now reads front-to-back for free.
- `radialTaper`/`TAPER` 0.84 → ~0.94 (wide flat base, no narrow foot). Remove `NOTCH_EXTRA` + the `LOBE_L/R` bulge system (they faked feet on a narrow taper); replace with a gentle superellipse-style squash on the lower hemisphere so the base reads flat without hard facets.
- Per-half asymmetry: side-keyed `fbm2` term (different seed per `sign(dir.x)`) nudging swell height a few % — kills the perfect mirror. Keep `undulate`/`wobble`.

**Ridge field — consolidate the three `atan` sites into one:**
- `ridgeFlow` returns `vec3(theta, warp, branch)`; `theta` becomes a blended flow angle: groove-parallel near the cleft → lateral wrap on the flanks, blended **as unit vectors** (never raw angles — ±π wrap seam), with `warp` added to theta so wandering/looping happens on the blended direction. `ridgeProfile`/`detailHeight`/frag AA all consume `flow.x` and delete their own `atan` calls. No new snoise/fbm calls (stays in the ~16-call/vertex budget); meander amplitude and `RIDGE_N` are eye-tuned.
**RD promotion (fragment-only — never in the displacement path):**
- Sample UV → dir-based equirect (`atan(dir.z,dir.x)`, `dir.y`), damped by existing `poleCapFade` to hide polar pinch.
- Thread `rd` into `detailHeight`'s fine-layer amplitude/crest mask (locally-constant across FD taps, like `finePhase`) for micro-relief; strengthen albedo modulation ±5% → ~±12–15% via `uRDBlend`. `ridgeProfile` untouched by RD (shared with vertex displacement; must stay stable during RD warmup). `uLook` stays 0 (crisp).

### WP3 — Atmosphere + pointer parallax (after WP2)
- `src/engine/pointer-parallax.ts` (new, pure, tested): critically-damped 2D spring step. ## Key risks
- Angle-blend seam at ±π → blend unit vectors, never raw angles.
- `lookAtQuaternion` NaN when forward ∥ up (elevated establish) → guarded + unit-tested.
- Parallax nudging camera into ridges at the finale → rotation-only offset.
- Spring never settling under demand frameloop → epsilon-gated invalidation.
- Keyboard scale-jump bypassing the intro lock → explicit phase guard.
- Perf: no new per-vertex noise calls; ≤3 extra draw calls (motes+shafts); re-profile with r3f-perf at dpr 2; dial icosahedron detail toward 48 only if a real device struggles.
