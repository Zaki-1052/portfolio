# Plan: HTML Fallback Polish — Atmospheric CSS & Component Refinement

## Context

Phase 1 built the complete semantic HTML layer: 6 scale sections, content pipeline, navigation. It's structurally correct but visually minimal — without the 3D Canvas, the site reads as unstyled text on a dark background. Each section is a flat `var(--bg)` rectangle with no visual rhythm, no atmospheric differentiation, and no sense of "descent."

The goal: make the no-WebGL version feel like a deliberately designed dark portfolio with warmth-to-cool atmosphere. Not a redesign — a CSS polish pass that adds atmospheric backgrounds, visual rhythm, refined component states, and enough personality that the fallback stands on its own. All changes must compose cleanly with the Phase 2 scroll engine and Phase 3+ 3D layers (the Canvas sits behind everything at `z-index: -1`, so CSS backgrounds on sections work fine).

## Approach: Three layers of polish

### 1. Atmospheric section backgrounds (globals.css)

Add CSS-only atmospheric effects to `ScaleSection` via `::before` and `::after` pseudo-elements. Each scale gets a subtle radial gradient and optional noise texture that creates the warm-to-cool descent without 3D.

**Per-scale atmospheric backgrounds via `[data-scale] section` styles:**
- `tissue`: Large warm radial gradient (gold-amber at 20-30% opacity) from upper-left, creating an organic glow. Slight warm vignette at edges.
- `cellular`: Smaller, cooler radial gradient (rose-magenta), positioned off-center. Less bloom than tissue.
- `chromatin`: Neutral — this is the Atom One Dark "home base." Very subtle blue radial at low opacity (~8-10%). The most understated section.
- `protein`: Cool-shifted radial gradient (cyan), subtle. Beginning of the digital feel.
- `code`: No radial glow. Instead, a faint CSS grid-line pattern or horizontal scanline texture via `repeating-linear-gradient`. Sharp, digital atmosphere.
- `expression`: Continues code register. Same grid pattern but even fainter.

**Implementation: a `[data-scale] section::before` rule per scale** that uses `radial-gradient` positioned with `background-position` to create non-centered, non-symmetrical atmospheric glows. The pseudo-element is `position: absolute; inset: 0; pointer-events: none; z-index: 0` behind content, with `opacity` controlling intensity. Content gets `position: relative; z-index: 1` to sit above.

**Section boundary treatment:** Add a `::after` pseudo-element at the bottom of each section that creates a soft gradient fade toward the next section's `--bg` color. This creates the "continuous descent" feeling between sections — a 60-80px gradient blend at each boundary using `linear-gradient(to bottom, transparent 0%, var(--bg-next, transparent) 100%)`. The `--bg-next` variable can be set per-section or handled with adjacent-sibling selectors.

**Film grain overlay (optional, very subtle):** A CSS noise texture via a tiny `data:` URI SVG pattern applied to `main::after`, at 2-4% opacity, that gives the warm scales a filmic quality and fades to 0 for code/expression. This is a single shared overlay, not per-section.

### 2. Component polish (globals.css + minor component edits)

**Extract recurring inline styles into CSS classes** where they repeat across multiple components. This isn't a full refactor — just the patterns that benefit from CSS-level control:

**New CSS classes to add to globals.css:**

- `.section-inner` — the max-width + margin wrapper inside ScaleSection (replace inline style)
- `.publication` — the left-border article pattern from ChromatinContent (already good, just formalize)
- `.status-aside` — the bordered box with key-value rows used in ProteinContent
- `.status-row` — the k/v flex row pattern used in ProteinContent (reusable for status boxes)
- `.featured-block` — the `$ cat` featured project blocks in CodeContent (sharp corners, deep bg, mono prompt)
- `.featured-grid` — 2-column grid that goes to 1-column on mobile (CodeContent featured projects need this responsive fix)

**ProjectCard refinements:**
- Remove `glass` prop (unused anywhere, adds dead code)
- Move hover state from JS `useState` to CSS `:hover` pseudo-class — eliminates React state for a pure CSS concern
- Add accent left-border on hover (1-2px, same pattern as branch buttons) as a subtle directional indicator
- Ensure cards stack well at narrow viewports

**Branch buttons (cellular):**
- Add `background` hover state: `var(--accent-soft)` on hover (currently only border changes)
- Active branch gets a subtle filled background: `var(--accent-soft)` persistent
- Transition the blurb text opacity separately (fade in more slowly)

**Contact rows (expression):**
- Add a left-border accent on hover (matching the publication left-accent pattern) — 1px that transitions from transparent to `var(--accent-line)`
- Key labels: slightly brighter on hover (from `--text-faint` toward `--text-muted`)

**Terminal listing (code):**
- Add `border-left: 2px solid var(--accent)` on hover rows (directional indicator)
- Format star counts with locale commas for readability (minor JS change in TerminalListing)

**Hero status box (tissue):**
- Add a subtle left-border accent (1px solid `var(--accent-line)`) — mirrors the publication pattern at the opposite end of the site
- Consider a faint radial gradient background inside the box (gold, 5-8% opacity)

**Depth indicator:**
- Add hover state on inactive dots: expand to 6-7px with transition
- Subtle track gradient (top: warm accent → bottom: cool accent) instead of flat color

### 3. Visual rhythm and micro-interactions (globals.css)

**Hairline separators:** Add a `<hr>` or CSS `::after` between major sections (not between tissue and cellular — the hero flows into branches — but between cellular/chromatin, chromatin/protein, etc.). Use `var(--border-soft)` with a max-width of ~120px, centered, to avoid the "full-width rule" template look.

**Focus indicators per scale:** The current `:focus-visible` ring uses `var(--accent)` which already adapts per scale. Verify this works on branch buttons, contact rows, and depth indicator dots. No code change likely needed, just verification.

**Link hover consistency:** Ensure all external links (project card titles, terminal listing names, publication titles if linked, contact links) show the `↗` indicator on hover for new-tab links. Currently only ProjectCard does this.

**Responsive fixes:**
- CodeContent featured grid: add `@media (max-width: 640px) { grid-template-columns: 1fr }` to stack on mobile
- ExpressionContent 2-column grid: stack on mobile
- ProteinContent 2-column grid: stack on mobile

## Files to modify

**Primary:**
- `src/styles/globals.css` — atmospheric backgrounds, new component classes, section rhythm, responsive fixes, micro-interactions

**Minor edits:**
- `src/components/ScaleSection.tsx` — add `::before` / `::after` support (ensure inner content div has `position: relative; z-index: 1` so pseudo-elements layer behind it)
- `src/components/ProjectCard.tsx` — remove `glass` prop, convert hover from `useState` to CSS `:hover`, add left-border accent
- `src/scales/code/TerminalListing.tsx` — format star counts with commas, convert hover from `useState` to CSS `:hover`
- `src/scales/code/CodeContent.tsx` — use `.featured-grid` class for responsive grid

**No structural changes to other components** — atmosphere comes from CSS `[data-scale]` selectors, not component markup.

## What NOT to do

- No JavaScript-driven animations or scroll listeners (that's Phase 2)
- No changes to content files or content pipeline
- No new React components
- No Tailwind utility classes for the atmospheric effects (CSS custom properties are the correct tool here)
- No `backdrop-filter: blur()` on atmospheric layers (performance concern and anti-slop rule)
- No `transform: scale()` hover effects anywhere
- No centered section headers or decorative blue underlines (anti-slop)
- No changes to the Canvas or 3D-related code
- No changes to the depth store or scroll engine setup
